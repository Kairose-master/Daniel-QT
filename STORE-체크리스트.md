## 1. 개인정보처리방침 공개 URL ✅ 문서 준비됨 / ☐ 호스팅 필요

`docs/privacy-policy.html`, `docs/terms.html` 를 만들어뒀습니다.
먼저 두 파일의 **`[여기에 실제 문의 이메일을 넣으세요]`** 를 실제 이메일로 바꾸세요.

### 무료로 공개 URL 만들기 (GitHub Pages, 권장)

이 프로젝트를 GitHub 저장소로 올린 뒤:

1. GitHub 저장소 → **Settings → Pages**
2. Source: **Deploy from a branch**, Branch: `main` / 폴더 `/docs` 선택 → Save
3. 몇 분 뒤 아래 주소가 생깁니다:
   ```
   https://<깃허브아이디>.github.io/<저장소이름>/privacy-policy.html
   https://<깃허브아이디>.github.io/<저장소이름>/terms.html
   ```

> GitHub 저장소가 없다면: 새 저장소 만들고 `git remote add origin <URL>` → `git push -u origin main`.
> 더 간단히: [netlify.com/drop](https://app.netlify.com/drop) 에 `docs` 폴더를 끌어다 놓으면 즉시 공개 URL이 나옵니다.

이 **privacy-policy.html URL** 을 Play Console 앱 콘텐츠 > 개인정보처리방침에 넣습니다.

## 2. 계정 삭제 ✅ 코드 있음 / ☐ 함수 배포 + Console 신고

앱 내 **내 설정 > 계정 삭제** 가 이미 있습니다. 서버 함수만 배포하세요:

```
npx supabase functions deploy delete-account
```

Play Console → **앱 콘텐츠 → 데이터 보안 → 계정 삭제** 에서:
- "앱에서 계정 삭제 가능" 선택
- 삭제 경로 설명: `앱 실행 > 우측 상단 프로필 > 내 설정 > 계정 삭제`

## 3. 프로덕션 빌드(AAB) ☐

```
npm run build:dev      # (아직이면) 개발 빌드로 기능 확인부터
eas build --profile production --platform android
```

빌드가 끝나면 `.aab` 파일 링크가 나옵니다. 이걸 Play Console 에 업로드합니다.

## 6. Play Console 양식 답안

### 데이터 보안(Data safety)
수집한다고 신고할 항목:
| 데이터 | 수집 | 목적 | 필수 |
|---|---|---|---|
| 이메일 주소 | 예 | 계정 관리 | 예 |
| 이름(별명) | 예 | 앱 기능(소그룹 표시) | 예 |
| 사용자 생성 콘텐츠(묵상 글·댓글) | 예 | 앱 기능 | 예 |
| 음성(오디오) | 예 | 앱 기능(응원 메시지) | 아니오 |
| 앱 활동/기타(출석) | 예 | 앱 기능 | 예 |

- 전송 중 암호화: 예 (HTTPS)
- 삭제 요청 가능: 예 (계정 삭제)
- 제3자 공유: 아니오 (위탁 처리만)

### 콘텐츠 등급 설문
- 폭력/성적 콘텐츠/약물 등: 모두 아니오
- 사용자 간 소통(UGC) 있음: 예 (댓글·음성) → 신고/차단 안내 필요할 수 있음
- 예상 등급: 전체이용가 / 3+

### 타겟 층
- 대상 연령: 만 13세 이상 권장 (사용자 간 소통 있음)
